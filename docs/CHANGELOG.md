## 产品概述

**小红书长文转图工作台** 是一款面向小红书长文创作者的 Next.js 在线工具，当前版本适配 Vercel Hobby 免费部署。

### 核心功能

| 功能 | 说明 |
|---|---|
| 富文本编辑 | 加粗、字号、颜色、换行、空格保留、emoji |

## v1.5.0 — 稳定性与防御性修复
> 迭代类型：稳定性修复 | 详细文档：[ITERATION-1.5.md](ITERATION-1.5.md)

### 概述

v1.5 是一次以稳定性为核心的修复迭代，重点补齐 API 输入校验、文档清洗防御、导出时序容错以及分页参数边界保护，降低异常输入和弱网场景下的 500 与导出失败风险。

### 变更内容

#### 1. API 输入校验升级为严格 schema

- 四个核心接口统一改为严格 `zod` schema + `safeParse`：
  - `POST /api/paginate`
  - `POST /api/editor/command`
  - `POST /api/suggestions`
  - `POST /api/compliance/check`
- 非法请求体统一返回 `422`，避免畸形结构漏入业务逻辑。

#### 2. 文档清洗增加运行时防御

- `sanitizeRichDoc` 由强假设输入改为防御式清洗。
- 对 `doc/nodes/children/marks` 做逐层守卫与默认值回退，避免因结构异常触发运行时崩溃。

#### 3. 命令请求链路补齐异常兜底

- 编辑器自然语言命令请求新增网络失败与 JSON 解析失败处理。
- 保持现有反馈机制，用户可获得明确错误提示，避免未处理 Promise 报错。

#### 4. 分页与序列化鲁棒性增强

- 图片尺寸与关键排版参数新增有限值校验和 clamp。
- 阻断 `NaN` 在分页计算链路中的传播，避免内容区负值或布局失稳。

#### 5. 导出挂载时序容错

- 导出前加入“下一帧短重试 + 完整性检查”。
- 页面节点尚未挂载完成时给出明确提示，不再直接空节点失败。

## v1.3.0 — 字号输入体验修复

> 迭代类型：可用性修复 | 详细文档：[ITERATION-1.3.md](ITERATION-1.3.md)

### 概述

v1.3 是一次针对编辑器排版控制可用性的修复迭代，聚焦字号手动输入体验的完整修复，使字号调节更精准、更直观。

### 变更内容

#### 1. 字号输入框手动输入修复

**问题描述：**

选中文本后，字号输入框无法进行手动删除与重新输入。原实现在 `onChange` 中加了范围限制（`v >= 10 && v <= 70`），导致用户清空原有数字时因值不合法而被直接拦截，输入过程无法继续。

**修复方案：**

引入独立的本地字符串状态 `fontSizeInput`，解耦显示值与实际应用值：

| 触发时机 | 行为 |
|---|---|
| `onChange` | 仅更新输入框显示，允许自由编辑 |
| `onFocus` | 锁定当前文本选区 + 全选输入框内容，方便快速覆盖输入 |
| `onBlur` 失焦 | 解析并应用字号；非法值时自动恢复为当前字号 |
| `Enter` 按键 | 立即解析并应用，等效于失焦确认 |

|---|---|
| `components/RichEditor.tsx` | 编辑器组件字号输入逻辑重构 |

## v1.0.0 — 初始版本

> 迭代类型：项目初始化

### 概述

v1.0 是小红书长文转图工作台的 Vercel 会话版初始版本，适配 Vercel Hobby 免费部署，提供面向小红书长文创作者的核心工具链。

### 核心功能

- **富文本编辑** — 支持加粗、字号、颜色、换行、空格保留
- **编辑区插图** — 支持上传 / 粘贴 / 拖拽方式插入图片
- **Mac 输入法 Emoji** — 直接在编辑器中输入并导出 emoji
- **对话式排版命令** — 自然语言排版指令解析
- **自动分页预览** — 按 1080 × 1440 尺寸自动分页

- **浏览器端导出发布包** — 支持导出 PNG 图集 + publish.md + meta.json + zip

### 技术特点

- 不保存历史项目，不连接数据库
- 图片在当前会话中以内嵌数据形式存在
- 刷新页面后文本和图片恢复至默认初始状态
- 导出在浏览器端完成，不依赖 Playwright 或服务端磁盘写入
